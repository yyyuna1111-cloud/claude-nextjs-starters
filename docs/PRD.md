# MLOps 대시보드 MVP PRD

## 핵심 정보

**목적**: 학습용 데이터/코드 업로드부터 Kubeflow 파이프라인 실행, GPU 학습 모니터링, 모델 아티팩트 저장 및 이력 추적까지 단일 대시보드에서 관리할 수 없는 불편함을 해소한다
**사용자**: Kubeflow 기반 ML 파이프라인을 운영하는 ML 엔지니어 및 데이터 사이언티스트

---

## 사용자 여정

1. 대시보드 홈 (로그인 후 자동 진입)
   ↓ 새 학습 시작 버튼 클릭

2. 학습 실행 페이지
   ↓ 데이터셋/코드 선택 → 파라미터 입력 → 학습 요청 제출

   데이터셋 없음 → 데이터셋/코드 관리 페이지 → 업로드 후 돌아옴
   데이터셋 있음 → 파라미터 입력 화면으로 진행
   ↓ 학습 요청 제출 완료

3. 파이프라인 모니터링 페이지 (자동 리디렉션)
   ↓ 파이프라인 단계 상태 실시간 확인 (prepare → train → evaluate → register)

   학습 실패 → 알림/장애 관리 페이지 → 원인 확인 후 재실행
   학습 성공 → 모델/아티팩트 관리 페이지 → 모델 등록
   ↓

4. 실험/이력 관리 페이지
   ↓ 이전 run과 파라미터/메트릭 비교

5. 모델/아티팩트 관리 페이지
   ↓ 모델 상태 변경 (Draft → Registered → Staging → Production)

6. 리소스/스토리지 현황 페이지 (독립 진입 가능)
   ↓ GPU/CPU 사용률 및 스토리지 상태 확인

7. 알림/장애 관리 페이지 (독립 진입 가능)
   ↓ 장애 알림 확인 → 해당 파이프라인/학습으로 이동 → 재실행

## 기능 명세

### 1. MVP 핵심 기능

| ID       | 기능명                      | 설명                                                                               | MVP 필수 이유                                           | 관련 페이지                                     |
| -------- | --------------------------- | ---------------------------------------------------------------------------------- | ------------------------------------------------------- | ----------------------------------------------- |
| **F001** | 학습 요청 생성              | 데이터셋/코드 선택, 파라미터 입력, CPU/GPU 모드 및 단일/멀티노드 선택 후 학습 제출 | 플랫폼의 핵심 진입점, 이 없이는 어떤 기능도 시작 불가   | 학습 실행 페이지                                |
| **F002** | 학습 실행 목록 조회         | 진행 중/완료/실패 학습 목록, 상태 필터링, 상세 보기 진입                           | 사용자가 자신의 학습 현황을 파악하는 기본 기능          | 학습 실행 페이지, 대시보드 홈 페이지            |
| **F003** | 데이터셋/코드 업로드        | 대용량 파일 업로드 (멀티파트), checksum 검증, 업로드 진행률 표시                   | 학습 데이터 없이 학습 요청 불가, 파이프라인 핵심 입력값 | 데이터셋/코드 관리 페이지                       |
| **F004** | 데이터셋/코드 버전 관리     | 버전별 파일 목록, 버전 비교, 특정 버전 선택                                        | 실험 재현성 보장을 위한 필수 기능                       | 데이터셋/코드 관리 페이지                       |
| **F005** | 파이프라인 단계별 상태 조회 | prepare/train/evaluate/register 각 단계의 실시간 상태 및 DAG 뷰                    | Kubeflow 파이프라인 진행 상황 파악, 장애 지점 식별      | 파이프라인 모니터링 페이지                      |
| **F006** | 실시간 로그 조회            | 학습 중 실시간 로그 스트리밍, 단계별 로그 필터링                                   | 학습 진행 상황 및 오류 원인 파악에 필수                 | 파이프라인 모니터링 페이지                      |
| **F007** | 실시간 메트릭 조회          | loss, accuracy 등 학습 메트릭 실시간 차트                                          | 학습 수렴 여부 조기 판단, GPU 낭비 방지                 | 파이프라인 모니터링 페이지                      |
| **F008** | 실험 이력 조회              | run 목록, 실행 시간, 상태, 파라미터 요약                                           | 과거 실험 기록 없이는 반복 실험 및 개선 불가            | 실험/이력 관리 페이지                           |
| **F009** | 파라미터/메트릭 비교        | 여러 run의 하이퍼파라미터 및 최종 메트릭 테이블 비교                               | 최적 하이퍼파라미터 선정을 위한 핵심 기능               | 실험/이력 관리 페이지                           |
| **F010** | 모델 아티팩트 조회          | 학습 결과 모델 파일, 체크포인트, 평가 결과 파일 목록 및 다운로드                   | 학습 결과물 확인 및 활용을 위한 필수 기능               | 모델/아티팩트 관리 페이지                       |
| **F011** | 모델 버전 관리 및 상태 변경 | 모델 등록, Draft/Registered/Staging/Production/Archived 상태 전환                  | 모델 배포 생명주기 관리, 운영 안전성 확보               | 모델/아티팩트 관리 페이지                       |
| **F012** | GPU/CPU 사용률 모니터링     | 노드별 GPU/CPU 실시간 사용률, 메모리 사용량                                        | 리소스 배분 최적화 및 병목 감지                         | 리소스/스토리지 현황 페이지, 대시보드 홈 페이지 |
| **F013** | 스토리지 현황 조회          | Rook-Ceph 스토리지 사용량, 데이터셋/모델별 용량 분석                               | 스토리지 부족으로 인한 학습 실패 사전 방지              | 리소스/스토리지 현황 페이지                     |
| **F014** | 멀티노드 상태 조회          | 클러스터 노드 목록, 각 노드의 상태 및 리소스 현황                                  | 멀티노드 학습 시 노드 장애 감지                         | 리소스/스토리지 현황 페이지                     |
| **F015** | 장애 알림 조회              | 학습 실패, GPU 부족, 스토리지 부족 알림 목록 및 상세 내용                          | 장애 인지 지연 방지, 빠른 대응을 위한 필수 기능         | 알림/장애 관리 페이지, 대시보드 홈 페이지       |
| **F016** | 학습 재실행                 | 실패한 학습을 동일 파라미터 또는 수정 파라미터로 재실행                            | 실패 복구 워크플로우의 핵심, 수동 재설정 시간 절감      | 알림/장애 관리 페이지, 학습 실행 페이지         |

├── 학습 실행
│   └── 기능: F001 (학습 요청 생성), F002 (학습 실행 목록), F016 (재실행)

├── 실험/이력 관리
│   └── 기능: F008 (실험 이력), F009 (파라미터/메트릭 비교)
├── 모델/아티팩트 관리
│   └── 기능: F010 (아티팩트 조회), F011 (버전/상태 관리)

│   └── 기능: F012 (GPU/CPU 모니터링), F013 (스토리지 현황), F014 (멀티노드 상태)

헤더 (우상단)
├── 미확인 알림 벨 아이콘 - F015

    ├── 내 프로필
    └── 로그아웃 - F020

비로그인 화면
├── 로그인 - F020
└── 회원가입 - F020

## 페이지별 상세 기능

### 로그인 페이지

> **구현 기능:** `F020` | **인증:** 불필요 (비로그인 접근)

| 항목            | 내용                                                                                                                 |
| --------------- | -------------------------------------------------------------------------------------------------------------------- |
| **역할**        | 서비스 진입을 위한 사용자 인증 전용 페이지                                                                           |
| **진입 경로**   | 비로그인 상태에서 모든 URL 접근 시 자동 리디렉션                                                                     |
| **사용자 행동** | 이메일/비밀번호 입력 후 로그인 제출, 회원가입 페이지 이동                                                            |
| **주요 기능**   | - 이메일/비밀번호 입력 폼 및 유효성 검사<br>- 로그인 오류 메시지 표시<br>- 회원가입 페이지 링크<br>- **로그인** 버튼 |
| **다음 이동**   | 성공 → 대시보드 홈 페이지, 실패 → 에러 메시지 표시                                                                   |

### 학습 실행 페이지

| --------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |

| **다음 이동**   | 제출 성공 → 파이프라인 모니터링 페이지 (해당 run), 파이프라인 보기 클릭 → 파이프라인 모니터링 페이지, 재실행 → 학습 요청 폼 파라미터 자동 채움                                                                                                                                                                                                             |
